Outline

        Below is a brief guide of ALTO test process. You will find following information in the guide:

        1.How to login to the start platform and what should be prepared by local and remote participants before the Interop day?

        2.The details of the test on interop-day.

    Preparation: Before Interop-day

        1.The interop volunteers have registered a IRC channel (#alto-interop@freenode). Please make sure you have an IRC client

          and be able to connect to the IRC channel before the interop-day.

        2.Each server participant should deploy its ALTO server with a public IP.

        3.Please download the test template from here

    Steps: On Interop-day

        Step 1. Start

        1.Please login to the IRC channel (#alto-interop@freenode), and each participant please announce yourself.

        2.Hearing all announcements, test chair will allocate a test ID (server A, .. client B) for every participant.

        3.Each server participant posts to the IRC its entry IRD.

        4.The test chair will continue to announce the matching of clients and servers.

        Step 2. Test

        1.Upon the announcement of a client-server matching, a google doc for the matching is created and announced on the IRC.

        2.The corresponding client participant will start to run its client to test the corresponding server.

        3.Test results will be recorded in the google doc. For a template of the good doc, please click here.

        4.After finishing testing, please post to the IRC to let the test chair know.

        Step 3. Summary of Results

        1.After all the tests finished, client participants should send test reports to the volunteers. Volunteers will summarize the results.

        2.The WG chair may report the summary results in the WG meeting next day.

ALTO in OpenDaylight

     We have implemented the basic version of ALTO in OpenDaylight. It already had these features:

1. Push and pull networkmap and costmap

2. Network Map Service

3. Cost Map Service

4. Filtered Network Map/ Cost Map Service

5. Endpoint Property Service

6. Endpoint Cost Service
